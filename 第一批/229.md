# [css] 第229天 在css中为什么说不建议使用@import？

- @import是在文档解析完成后才加载css文件，他会延迟页面的渲染。link标签会在文档解析时并行下载文件不会阻塞页面的渲染。
- @import的下载是串行的，下一个@import必须等待前一个@import下载完成才能开始下载。
- 多个@import语句的下载顺序是不确定的
- 老浏览器不兼容
- 难以维护

# [vue] vue能监听到数组变化的方法有哪些？为什么这些方法能监听到呢？

push、pop、shift、unshift、splice、sort、reverse

vue对这些方法进行了重写，在保留原有功能的基础上，添加了watcher，从而实现对数组的监听。

# [NodeJs] 第1292天 请讲讲libuv线程池运行原理 

- 默认配置了一个固定大小的线程池，通常是cpu核心数-1，这样做是避免过多的上下文切换开销
- 当需要执行任务时，任务会被传递给libuv的线程池，主线程将任务放入一个任务队列，然后由线程池中的空闲线程来执行
- 当io操作完成后，工作线程会通过ipc通知主线程，主线程再通过事件循环来处理事件
- 考虑到资源的有效管理和优化，libuv限制了并发执行的文件操作数量，也提供了一定程度的负载均衡，确保线程池中的线程能均匀的分摊任务。
